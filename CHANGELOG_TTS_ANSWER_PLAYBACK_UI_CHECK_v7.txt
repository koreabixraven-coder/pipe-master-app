배관기능장 앱 v7 수정 기록

사용자 보고 증상.
- 문제를 읽어주는 중 정답 안내가 '정답 4번'이 아니라 '정답 4번번'으로 들림.
- 보기 사이 쉬는 시간 0.5초 옵션과 충돌이 의심됨.
- 전체 UI. 문제 목차. 이론 목차. 재생. 정지 상태 점검 요청.

원인.
- 보기 읽기 방식의 기본 라벨이 '1번. 2번. 3번. 4번' 형태인데,
  기존 spkAns 함수가 여기에 다시 '번'을 붙여 '4번번'으로 낭독했음.
- 보기 사이 쉬는 시간 0.5초 옵션이 직접 원인은 아니지만,
  일시정지 중 쉬는 시간 타이머가 끝나면 다음 보기로 이어지지 않을 수 있는 잠재 문제가 있었음.
- 이론 화면의 정지 버튼이 ttsStop()을 호출하지만 실제 함수 정의가 없어 이론 낭독 정지에 문제가 생길 수 있었음.
- serviceWorker 등록 경로가 /sw.js 절대경로로 남아 GitHub Pages 하위 경로에서 불안정할 수 있었음.

수정.
1. 정답 TTS는 보기 읽기 방식과 분리하여 항상 '정답은 N번입니다.'로 낭독하도록 수정.
2. '정답은 4번번' 중복 낭독 제거.
3. 보기 사이 쉬는 시간 중 일시정지하면 재개 후 이어서 읽도록 resumeTask 보정.
4. 음성 낭독 종료 시 일시정지 상태이면 다음 동작을 버리지 않고 재개 대기 상태로 저장.
5. 이동 중 청취 모드 일시정지는 생각 타이머를 강제 종료하지 않도록 수정.
6. 수동/자동 정답 표시에도 '정답: N번. 보기내용' 형태로 명확화.
7. 이론 낭독 정지용 ttsStop() 함수 추가.
8. serviceWorker 등록 경로를 sw.js 상대경로로 수정.
9. sw.js 캐시명을 v7로 변경.
10. 문제 목차 16개 × 100문제 = 1600문제 정적 점검 완료.
11. 이론 목차 8개 과목. 21개 챕터. 65개 섹션 정적 점검 완료.
